The Comprehensive Guide to Core Web Vitals Optimization

SY
Shihad Y Lead Web Architect • July 4, 2026

In today's search landscape, speed is no longer just a technical recommendation—it is a core ranking factor. Google's Page Experience update uses three key metrics, known as Core Web Vitals, to evaluate page speed. Optimizing these metrics is essential to securing top rankings in search results.

1. Largest Contentful Paint (LCP)

LCP measures perceived load speed. It marks the point in the page load timeline when the primary content (typically a hero image or main heading block) has likely loaded. An LCP under 2.5 seconds is considered good.

How to Optimize LCP:

  • Preload Critical Assets: Use `rel="preload"` tags to prompt browsers to download the main hero image or stylesheet immediately.
  • Compress Images: Use modern file formats like WebP or AVIF, and resize images appropriately for mobile screens.
  • Minimize Blocking CSS: Inline essential styles and defer non-critical layout sheets.

2. Cumulative Layout Shift (CLS)

CLS measures visual stability. It quantifies how much elements on a page move around while loading, which can lead to users accidentally clicking the wrong buttons on mobile screens. A good CLS score is under 0.1.

How to Optimize CLS:

  • Set Dimensions: Always specify width and height values on images and video wrappers so browsers can reserve space for them during layout rendering.
  • Avoid Late-Injected Content: Do not insert dynamic banners or ads above existing content without allocating container space beforehand.
  • Use CSS transform: Use CSS transforms for animations instead of altering top, left, or margin values, which trigger browser reflows.

3. Interaction to Next Paint (INP)

Replacing First Input Delay (FID), INP evaluates overall responsiveness. It measures the latency of all user interactions (clicks, taps, keyboard presses) throughout their visit. A good INP score is under 200 milliseconds.

How to Optimize INP:

  • Defer JavaScript Execution: Use `async` or `defer` attributes on script tags to keep script evaluation from blocking the main thread.
  • Split Long Tasks: Break down intensive calculations into smaller chunks using requestAnimationFrame.
  • Avoid DOM Bloat: Keep your total DOM tree node count below 1,000 to minimize browser recalculation times.
